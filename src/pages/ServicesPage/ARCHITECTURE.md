## Key Design Patterns Used

1. **Container/Presenter Pattern**
   - ServicesPage: Container (logic)
   - Components: Presenters (UI)

2. **Custom Hook Pattern**
   - useBookingFlow: Encapsulated logic
   - Reusable, testable

3. **Composition Pattern**
   - Small components compose larger ones
   - BookingModal = ProgressSteps + Steps

4. **Factory Pattern**
   - formatServicesForUI: Transforms data
   - Consistent interface

5. **Observer Pattern**
   - React state updates
   - Components re-render on change

6. **Strategy Pattern**
   - Different steps in booking flow
   - Swappable implementations
